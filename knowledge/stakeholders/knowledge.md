# Stakeholder Knowledge

Qualitative knowledge about how specific stakeholders think, communicate, and make decisions.
Written by the agent when it observes a consistent preference (after 2+ signals), or added directly by the user via chat ("Remember that I prefer…").

This is distinct from `knowledge_beliefs` (data-driven findings about the world). Stakeholder preferences are editorial and personal — they belong here, not in the beliefs table.

---

## Format

Each entry follows the `## [Name / Role]` heading pattern. Entries are added or updated by the agent when a new preference signal is observed.

### Preference categories

| Category | Examples |
|----------|---------|
| **Communication style** | "Prefers direct answers, not summaries", "Always wants the headline number first" |
| **Analytical preferences** | "Prefers weekly comparisons over daily", "Always asks about engagement rate first" |
| **Approval patterns** | "Wants to approve all code before execution", "Happy to auto-approve analysis under 10 lines" |
| **Focus areas** | "Prioritizes dwell time over traffic volume", "Always benchmarks against Q4 holiday" |
| **Trust level** | "Skeptical of agent conclusions without cited data", "Trusts agent recommendations on chart type" |
